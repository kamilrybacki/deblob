# Deblob P2-D (Semantic Fingerprint) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add the third identity dimension — a deterministic `sem_` fingerprint over controlled-vocabulary field metadata — so byte-identical structures with different meaning get distinct identity, without changing the wire tag, without SLM auto-application, and without ever causing a merge.

**Architecture:** A new pure `deblob-semantic` crate owns the `deblob-semantic-v1` vocabulary tables, validation, canonical serialization, and the digest. `deblob-core` gains the `sem_` id + metadata types + two optional fields on `SchemaRecord`. `deblob-redis` stores the semantic block on the immutable schema hash via the existing atomic Lua path (byte-compare immutability, non-destructive migration). The `deblob` bin exposes an authenticated+audited governance API and a computed (not-acted-on) semantic-drift signal. Un-annotated schemas are byte-identical to today.

**Tech stack:** Rust; `sha256`/`data-encoding` (reuse the `digest_id!` macro), `serde` with `deny_unknown_fields`, the existing `deblob-*` crates, `redis`. No new heavy deps.

**Spec:** `docs/superpowers/specs/2026-07-15-deblob-p2d-semantic-fingerprint.md` — referenced by § below.

## Global constraints

- **`sem_` NEVER rides on the wire in P2-D.** No `deblob-semantic-id` header. Message tagging stays `sch_`/`cand_`. A test must assert the wire tag is unchanged by annotation.
- **Never free prose.** Every stored/hashed semantic attribute is a registered, versioned vocabulary token. No free-string field exists in `FieldSemantics`.
- **Un-annotated = true `Option`/`None`** (per Hermes review §3), NEVER a sentinel: no `sem_none`, no all-zero digest, no hash-of-empty, no reverse-index entry, no equality inference between two un-annotated schemas. A `sem_` is created only when ≥1 canonical assertion survives normalization. `privacy_class` is governance, NOT part of the intrinsic `sem_`.
- **Only ever ADD a split.** The semantic fingerprint can never cause two families to merge. Both diagnostics (drift; same-`sem_`/diff-structure) are one-directional, proposal-only.
- **Operator-gated, like promotion.** Semantic metadata enters only via the authenticated + audited management API. Producer-supplied semantic input on the ingest path is stripped. The SLM (shadow) only proposes; it never applies.
- **No auto family-split/merge.** Diagnostics are computed + surfaced only. Acting on them is the P3/P4 live gate.
- **Append-only revisions, not schema-record mutation** (per Hermes review §4). The immutable `sch_` record is UNTOUCHED by annotation; semantics live in immutable append-only revisions + a mutable active pointer (ETag). Pre-P2-D schemas read back `None`; `rebuild_index` restores the reverse index from active pointers.
- **Off by default / no-op.** Un-annotated behavior is byte-identical; every existing test stays green.
- Domain separation: `sem_` uses `deblob-semantic-v1`, distinct from `sch_`'s `deblob-canon-v1`; a `sem_` can never be parsed as a `sch_`.
- TDD; 80%+ coverage; deterministic golden tests for the digest.

### Task 3: canonical serialization + the `sem_` digest

**Files:** create `crates/deblob-semantic/src/{canon.rs, digest.rs}`; `crates/deblob-semantic/tests/digest_it.rs`.

**Produces:** (per `deblob-p2d-hermes-review.md` §2/§3 — a byte-level protocol, NOT generic JSON hashing) `canonical_semantic_bytes(&SemanticMetadata) -> Vec<u8>`:
- Encode each path as typed segments; NFC-normalize key segments; reject invalid surrogates/NUL/control chars; no case folding; detect duplicate paths after normalization (error); the wildcard is a distinct typed segment.
- Sort field entries by their canonical encoded path bytes. Emit `event_type` (if present) in a fixed schema-level position.
- Fixed attribute order per the protocol; a missing attribute is absent; an explicit `null` is invalid; an empty nested struct normalizes to absent; a field entry with no attributes is removed.
- Enum entries sorted by canonical **typed value bytes** (`integer 1` ≠ `string "1"`; canonicalize numeric enum values via the P1 numeric protocol); meaning codes carry `{vocabulary, code}`.
- Numbers (`numeric_scale`, epoch) as signed canonical integers / canonical decimal strings — NEVER float.
- No defaults in v1.

`semantic_fingerprint(&SemanticMetadata) -> Option<SemanticFingerprint>` — if NO assertion survives normalization (every field empty and no `event_type`) → `None` (no bytes, no hash); else `Some(SemanticFingerprint(SemanticId::from_digest(&sha256("deblob-semantic-v1\0" ++ canonical_semantic_bytes))))`. Return the canonical bytes too (stored for byte-compare, Task 5). `sch_` is NEVER in the preimage.

- [ ] Write failing tests (goldens): determinism — same metadata → same `sem_` across calls and across field-entry input order; **sensitivity** — `unit` `Cel`→`[degF]` on an otherwise identical map → *different* `sem_`; changing any single attribute (namespace, event_type, numeric_scale, one enum-meaning entry, a path segment, temporal.resolution s→ms) → different `sem_`; a field literally named `"a.b"` (one Key segment) ≠ path `a`→`b` (two segments) — the typed-path anti-ambiguity case; duplicate normalized paths → error; **None** — a metadata with no surviving assertion → `None` (NOT the hash of empty bytes, NOT a sentinel); domain separation — a `sem_` never parses as a `sch_`; `sch_` absent from the preimage (two different `sch_` with identical semantics → SAME `sem_`).
- [ ] Run → implement → run.
- [ ] Commit `feat(semantic): byte-level canonical protocol + sem_ digest (typed paths, no-float, empty → None)`.

Reviewer checkpoint — this is the identity core; determinism/sensitivity/typed-paths/None are load-bearing.

### Task 5: vault storage — append-only semantic revisions + active pointer + reverse index

> **AMENDED per `deblob-p2d-hermes-review.md` §4 (authoritative).** Semantics are NOT a mutable block on the immutable `sch_` record. They are **append-only immutable revisions** with a mutable transactional active pointer, so `sch_` stays immutable AND meaning stays correctable.

**Files:** `crates/deblob-redis/src/registry.rs`, `crates/deblob-redis/src/lua.rs`; reuse the audit + rebuild paths.

**Produces:** an append-only revisions store + active pointer:
- `deblob:sem-rev:<sch_id>:<revision_id>` (immutable): `{ revision_id, sch_id, sem_id, canonical_semantic_bytes, previous_revision_id, actor, reason_code, reason, recorded_at, effective_from, status }`.
- `deblob:sem-active:<sch_id>` (mutable pointer): the active `revision_id` + its `sem_id` + a monotonically-increasing ETag.
- `deblob:sem-index:<sem_id>` (set): reverse `sem_ → {sch_id}` — populated ONLY for real `sem_` (never for `None`).
- `append_revision(sch_id, metadata, canonical_bytes, sem_id, actor, reason_code, reason, effective_from, expected_etag) -> Result<Revision, SemError>` in **one atomic Lua transition** (append immutable revision + advance active pointer + update reverse index + audit event, or nothing). Semantics: identical `canonical_bytes` to the active revision → idempotent no-op (`AlreadyActive`); different bytes + missing `reason` → `MissingReason`; different bytes + `expected_etag` ≠ current → `EtagConflict`; different bytes + reason + correct etag → append + advance. `reason_code` ∈ the controlled enum. Never delete/overwrite a prior revision.
- `active_semantic(sch_id) -> Option<(SemanticMetadata, SemanticId, Etag)>` (None if never annotated — no sentinel). `revisions(sch_id) -> Vec<Revision>` (full history). `schemas_by_semantic(sem_id) -> Vec<SchemaId>`.
- Migration: a schema with no `sem-active` key reads back `None`. `rebuild_index` restores the reverse index from the active pointers (revisions are the source of truth). The `sch_` schema record is UNTOUCHED by annotation (immutability preserved by construction).
- `effective_from` is STORED; P2-D does NOT build the ingestion-time-aware resolver (no consumer — documented P4).

- [ ] Write failing tests (vs a real Redis, mirroring the registry IT harness): append a revision → `active_semantic` returns metadata + `sem_` + etag; `schemas_by_semantic` finds it; re-append identical bytes → idempotent `AlreadyActive`, no new revision; different bytes without reason → `MissingReason`; different bytes with stale etag → `EtagConflict`; different bytes + reason + correct etag → new immutable revision, pointer advanced, prior revision still readable via `revisions()`, reverse index updated (old `sem_` de-linked, new linked); a schema never annotated → `None`; after `rebuild_index`, the reverse index is restored; the `sch_` schema record bytes are unchanged after annotation.
- [ ] Run → implement → run.
- [ ] Commit `feat(redis): append-only semantic revisions + active pointer (ETag) + sem_ reverse index`.

Reviewer checkpoint — revision immutability + atomic pointer advance + migration are the corruption surface.

### Task 7: semantic-drift signal (computed, not acted on)

**Files:** create `crates/deblob/src/semantic_drift.rs`; wire into the family-version + metrics paths.

**Produces:** (per `deblob-p2d-hermes-review.md` §5) TWO one-directional, proposal-only diagnostics — neither ever aliases/merges/mutates a family:

**(a) Semantic drift** — when a family gains a **structurally-compatible** new version whose active `sem_` differs from the prior version's, record `SemanticDrift { family_id, prior_version, new_version, prior_sem, new_sem }`: counter `deblob_semantic_drift_total` + a family-version field + an API entry. Does NOT split. `None`→`Some` (first annotation) is NOT a drift (documented).

**(b) Same-`sem_` / different-`sch_`** — from the reverse index, when one `sem_` maps to ≥2 schemas, emit `same_semantic_fingerprint_different_structure` and classify deterministically by structural compatibility (compatible → possible rename/false-split; incompatible → representation change / bad annotation / drift; identical paths + changed types → high-value review). Assign a **strength** using annotation coverage: `strong` = same `canonical_event_type_id` AND ≥80% of leaf fields carry a `canonical_field_id` (semantic-group agreement is P4); `medium` = same event type + partial coverage; `weak` = only sparse unit/namespace/enum overlap. Only `strong`/`medium` create a review candidate; `weak` is logged (counter `deblob_semantic_collision_total{strength}`) but never a family link. Diagnostic HINT only — sparse identical annotations do not prove equivalence.

- [ ] Write failing tests: two compatible versions same `sem_` → no drift; different `sem_` → counter increments, record emitted, family **not** split (version count/family id unchanged); `None`→`Some` → no drift; one `sem_` under two schemas with high coverage + same event_type → `strong` review candidate; under two schemas with only a shared unit → `weak`, logged not candidate; no path produces an alias/merge/family mutation.
- [ ] Run → implement → run.
- [ ] Commit `feat(bin): semantic-drift + same-sem_/diff-structure diagnostics (strength-classified, proposal-only, never auto-splits)`.

### Task 9: path-independent semantic signature + similarity neighbor search

> **NEW (user request, mid-P2-D).** Find schemas *semantically SIMILAR* to each other (not only exact-same `sem_`) — the "vector-DB-like" neighbor search. This pulls Hermes' deferred **path-independent semantic signature** forward from P4. It is DETERMINISTIC (controlled-vocabulary features, NO learned embeddings — those stay P4) and strictly **diagnostic** (never merges/aliases). May be refined by Hermes' `deblob-p2d-02` answer — fold it before dispatch if returned.

**Files:** `crates/deblob-semantic/src/signature.rs` (the signature + metric); `crates/deblob-redis/src/registry.rs` (a bounded signature index); `crates/deblob/src/api/` (the neighbors endpoint).

**Produces:**
- `semantic_signature(&SemanticMetadata) -> SemanticSignature` — a deterministic, **path-independent** sparse feature multiset over controlled-vocab ONLY: `{ event_type }` ∪ per-field `{ canonical_field_id, unit(system+code), identifier_namespace, meaning-code vocabulary, temporal.kind }`. It ignores field PATHS/NAMES (so a pure rename stays near) and raw values. No feature derived from free text.
- `similarity(&SemanticSignature, &SemanticSignature) -> f64` — **weighted Jaccard** over the feature multiset (weights: `event_type` and `canonical_field_id` dominate; sparse unit/namespace/temporal overlap contributes less — mirrors the strong/medium/weak tiers). Deterministic; no float hashing.
- A **bounded inverted index** `deblob:sem-sig:<feature>` (feature → schema set) maintained alongside the reverse `sem_` index (Task 5), so `neighbors(sch_id, k)` gathers only schemas sharing ≥1 feature (candidate set) then ranks by `similarity` — never a global scan (mirrors the P1 structural bucket index). Optional MinHash-LSH banding only if the candidate set proves too large (documented; not required at homelab scale).
- API: `GET /api/v1/schemas/{sch_id}/semantic-neighbors?k=` (management port, authenticated) → top-k similar schemas + scores, **diagnostic only** (a header/field documents "proposal, not a merge").

- [ ] Write failing tests: two schemas with the SAME field meanings but DIFFERENT field paths/names (a pure rename) → high similarity + returned as neighbors (the headline case — path-independence); two unrelated schemas → low/zero similarity, excluded; `event_type` + `canonical_field_id` agreement scores higher than a shared unit alone (weighting); `neighbors` gathers via the inverted index (candidate set = only feature-sharing schemas, asserted bounded, not a full scan); an un-annotated schema (`None` signature) has no neighbors and no index entries; the endpoint never mutates/aliases/merges.
- [ ] Run → implement → run.
- [ ] Commit `feat(semantic): deterministic path-independent semantic signature + bounded similarity neighbor search (diagnostic-only, no embeddings)`.
