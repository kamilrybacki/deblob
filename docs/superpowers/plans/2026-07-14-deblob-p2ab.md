# Deblob P2-A/B (SLM Lane + Eval Harness) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add the provider-agnostic SLM discovery lane in SHADOW mode (`HttpInferencer` default) plus an offline eval harness, so novel structural clusters get a *proposed* classification (match/new/abstain) that is logged and measured but never applied.

**Architecture:** New `deblob-slm` crate implements the P1 `SemanticInferencer` port with `HttpInferencer` (OpenAI-compatible). A shadow classifier in the `deblob` bin invokes it per stable candidate cluster after deterministic top-k retrieval over the P1 bucketed index; decisions land in a shadow log, nothing is applied. New `deblob-eval` crate scores a configured endpoint against a golden corpus, reporting wrong-valid rate separately.

**Tech stack:** Rust; `reqwest` (HTTP client), `serde`/`serde_json`, `wiremock` (mock endpoint in tests), the existing `deblob-*` crates, `redis` (shadow log stream). No real model in CI.

**Spec:** `docs/superpowers/specs/2026-07-14-deblob-p2a-slm-lane.md` — referenced by § below.

## Global constraints

- SHADOW ONLY: no SLM decision is ever applied to registry/index/candidate state in P2. A shadow-wiring test MUST assert zero state mutation.
- Provider-agnostic: all model access via the `SemanticInferencer` port; nothing above it knows the vendor.
- 3-way contract: `match_schema(id, exact|compatible_drift|incompatible_similarity) / new_candidate(reason) / abstain(reason)`; `id` ∈ retrieved top-k only; enum reasons (not prose); one mechanical repair then abstain.
- PII-safe: prompts carry monoid stats + redacted, length-capped, injection-checked field NAMES only — never raw payload values. Redaction fails closed.
- Prefill economics: retrieve top-k (≤10), never attach the full registry; cache prompt prefix + any grammar by candidate-set digest.
- Endpoint unavailability/timeout = a shadow "unavailable" outcome, never a cold-lane/relay failure.
- `DEBLOB_SLM_API_TOKEN` env-only, never logged, never in TOML.
- TDD; `wiremock` for the endpoint; 80%+ coverage.

### Task 1: `deblob-slm` scaffold + 3-way contract types

**Files:** create `crates/deblob-slm/{Cargo.toml, src/lib.rs, src/contract.rs}`; add to workspace.

**Produces:** `InferenceRequest { candidate: CandidateProfileView, retrieved: Vec<FamilyCandidate>, contract_version: u32, budget }`; `InferenceDecision` enum (`MatchSchema{family_id, relation, evidence}` / `NewCandidate{reason}` / `Abstain{reason}`); `Relation` (`Exact|CompatibleDrift|IncompatibleSimilarity`); bounded `NewReason`/`AbstainReason` enums; `InferenceError`. A `validate_decision(decision, allowed_ids) -> Result<InferenceDecision, ContractError>` that rejects: any `family_id` not in `allowed_ids`, extra/unknown fields (serde `deny_unknown_fields`), free-prose where an enum is required.

- [ ] Write failing tests: a MatchSchema naming an unlisted id → `ContractError::IdNotAllowed`; unknown JSON field → rejected; each enum round-trips snake_case.
- [ ] Run (fail) → implement types + validate_decision → run (pass).
- [ ] Commit `feat(slm): 3-way inference contract types with id-allow-list validation`.

*(AMEND per Hermes Q1: contract granularity / missing fields for shadow measurement.)*

### Task 2: `HttpInferencer` against an OpenAI-compatible endpoint

**Files:** create `crates/deblob-slm/src/http.rs`, `src/cache.rs`; `tests/http_it.rs` (wiremock).

**Produces:** `HttpInferencer::new(cfg: SlmHttpConfig)` implementing `SemanticInferencer`; builds a structured-output/tool-calling request for the fixed 3-way contract, parses the response into `InferenceDecision`, runs `validate_decision`, does ONE mechanical repair on a malformed-but-recoverable response then `Abstain`, and caches decisions by `(model_digest, contract_version, candidate_set_digest)`. Timeout + bounded concurrency; a transport/timeout error → `InferenceError` (mapped by the caller to a shadow "unavailable").

- [ ] wiremock tests: a well-formed endpoint response → the expected decision; a response naming an unlisted id → one repair → Abstain; a timeout → InferenceError; identical request twice → second served from cache (assert endpoint hit once).
- [ ] Run → implement → run.
- [ ] Commit `feat(slm): HttpInferencer (OpenAI-compatible) with structured output, one-repair, decision cache`.

*(AMEND per Hermes Q2: structured-output vs GBNF; constraint-tax avoidance / "reason free, constrain late".)*

- [ ] Tests (fake registry seeded with families): the closest family ranks first; an unrelated family is excluded or ranked last; k is honored; the retrieved set feeds `FamilyCandidate` shape.
- [ ] Run → implement → run.
- [ ] Commit `feat(bin): structural-distance top-k retrieval over the bucketed index`.

### Task 4: PII-safe prompt builder + injection detection

**Files:** create `crates/deblob-slm/src/prompt.rs`.

**Produces:** `build_prompt(request) -> Prompt` emitting monoid statistics + redacted field NAMES (length-capped, escaped as data) + the top-k family summaries + the allowed id list. `detect_injection(field_name) -> bool` flags instruction-like / control-token sequences. A hard invariant: NO raw payload value can appear in the output.

- [ ] Tests: a candidate carrying a sensitive literal value → that literal NEVER appears in the built prompt (assert absent); an injection-y field name (e.g. "ignore previous instructions") → flagged; field names over the cap are truncated/escaped.
- [ ] Run → implement → run.
- [ ] Commit `feat(slm): PII-safe prompt builder with field-name injection detection`.

### Task 5: Shadow wiring + shadow decision log

**Files:** create `crates/deblob/src/shadow.rs`; wire into the cold lane; add a shadow-log store (Redis stream `deblob:shadow:<candidate>`, trimmed).

**Produces:** a shadow classifier invoked once per STABLE candidate cluster (debounced) that: retrieves top-k, builds the prompt, calls the `SemanticInferencer`, evaluates policy (retrieval margin, producer consistency, sample coverage, deterministic compatibility, model agreement — NOT model self-confidence), and appends a `ShadowDecision { candidate_set_digest, retrieved_ids+distances, deterministic_baseline_verdict, model_decision, contract_version, model_digest, policy_result, endpoint_status }` to the log. APPLIES NOTHING.

- [ ] Integration test (fake `SemanticInferencer`): after a shadow run, assert (a) a ShadowDecision was logged with the expected fields, and (b) ZERO mutation of schema/candidate/index state (shadow means shadow). Endpoint-unavailable → a ShadowDecision with `endpoint_status = unavailable`, cold lane unaffected.
- [ ] Run → implement → run.
- [ ] Commit `feat(bin): shadow classifier + decision log (proposes and logs, applies nothing)`.

*(AMEND per Hermes Q4: exact shadow-log fields + the go-live risk–coverage criterion.)*
