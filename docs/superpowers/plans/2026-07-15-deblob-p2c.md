# Deblob P2-C (HTTP Push Reverse Proxy) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add an HTTP push reverse proxy that tags forwarded JSON payloads with the same deterministic core as the Kafka relay, forwards to a fixed upstream allowlist, feeds unknown shapes to the discovery lane, and is hardened per the checklist. Disabled by default.

**Architecture:** New `deblob-http` crate: axum server that on POST enforces limits → strips reserved+hop-by-hop headers → classifies the body via the shared `HotMatcher` → tags → forwards the unmodified body to an allowlisted upstream → returns the response with the tag. Wired into `serve()` behind `[http_proxy].enabled` (default false).

**Tech stack:** Rust; `axum` 0.8 + `hyper`/`tower` (proxy), `reqwest` (forward client), the existing `deblob-match` (HotMatcher, DiscoveryMsg), `deblob-kafka` (discovery producer), `deblob-redis`. `wiremock`/axum stub as the test upstream.

**Spec:** `docs/superpowers/specs/2026-07-15-deblob-p2c-http-proxy.md` — referenced by § below.

## Global constraints

- Payload bytes NEVER mutated; tags ride in headers; exactly one `deblob-schema-id` + one `deblob-origin` on the forwarded request; ALL inbound `deblob-*` headers stripped first (§4).
- Fixed upstream allowlist — forward destination NEVER from a client-controlled header/path (SSRF prevention, §4).
- Malformed body → 422 + `deblob-quarantine-reason`, NOT forwarded upstream (§6).
- Registry down → tag `unresolved` (never `cand_`), still forward (§6).
- `[http_proxy].enabled=false` is the default and MUST leave existing behavior + all tests unchanged.
- Ingest auth secret (if any) env-only, never logged.
- TDD; reuse `HotMatcher` — no new schema-identity logic; 80%+ coverage.

### Task 1: `deblob-http` scaffold + tag-and-forward core + header hygiene

**Files:** create `crates/deblob-http/{Cargo.toml, src/lib.rs, src/proxy.rs, src/headers.rs}`; add to workspace; `tests/proxy_it.rs`.

**Produces:** `HttpProxy::run(cfg: HttpProxyCfg, matcher: Arc<HotMatcher>, discovery: Option<Arc<dyn DiscoverySink>>, shutdown: CancellationToken)`. `HttpProxyCfg { listen_addr, upstream_allowlist, route, limits }`. `headers.rs`: `strip_reserved_and_hop_by_hop(headers) -> HeaderMap` (strip case-insensitive `deblob-*` + `Connection`/`Keep-Alive`/`Transfer-Encoding`/`TE`/`Trailer`/`Upgrade`/`Proxy-*`), `with_tag(headers, &SchemaRef, origin)`. A `DiscoverySink` trait (`async fn enqueue(&self, DiscoveryMsg)`) so Task 3 can back it with Kafka or a fake.

Core POST handler: read bounded body → `matcher.classify(body, limits)` → on Known/Unknown attach `deblob-schema-id` + `deblob-origin`, forward the UNMODIFIED body to the allowlisted upstream (reqwest), return the upstream response with the tag header added.

- [ ] Write failing integration test `known_shape_tagged_and_forwarded` (spin an axum test upstream on the allowlist; POST a known-ish JSON; assert the upstream received the body + a `deblob-schema-id` header, and the client got the response). Plus a unit test `strips_reserved_and_hop_by_hop` (a request with `Deblob-Schema-Id: forged` + `Connection: keep-alive` → both stripped before forward).
- [ ] Run (fail) → implement → run (pass).
- [ ] Commit `feat(http): reverse-proxy scaffold with tag-and-forward + header hygiene`.

### Task 2: hardening (limits, allowlist, malformed, smuggling, timeouts)

**Files:** `crates/deblob-http/src/limits.rs`; extend proxy.rs + tests.

**Produces:** enforcement of every §4/§6 rule.

- [ ] Tests (each a rule): `oversize_body_413` (body > max_body_bytes → 413, upstream NOT called — assert via a never-hit upstream); `malformed_422_not_forwarded` (duplicate-key/NaN body → 422 + `deblob-quarantine-reason`, upstream not called); `disallowed_upstream_rejected` (a route/target not in the allowlist → 404/502, never forwarded); `cl_and_te_rejected` (both Content-Length and Transfer-Encoding, or duplicate CL → 400); `header_size_capped`; `read_timeout` (a slow-body client → bounded timeout, not a hang). Limits enforced BEFORE reading the full body where possible (Content-Length precheck + streamed cap).
- [ ] Run → implement → run.
- [ ] Commit `feat(http): hardening — body/header limits, allowlist, malformed 422, smuggling + timeout defenses`.

### Task 3: discovery feed + idempotency + outage behavior

**Files:** extend proxy.rs; a Kafka-backed `DiscoverySink` (in deblob-http or deblob-kafka) reusing the relay's discovery producer; tests.

**Produces:** unknown-shape → `cand_` + a `DiscoveryMsg` enqueued to the `DiscoverySink` (so the cold lane sees HTTP unknowns); `Idempotency-Key` accepted-or-generated and forwarded downstream; registry-down → `unresolved` still forwarded.

- [ ] Tests: `unknown_shape_enqueues_discovery` (unknown JSON → tagged `cand_`, and a fake DiscoverySink received a DiscoveryMsg with the right source/cursor-analogue); `idempotency_key_forwarded` (a client key is preserved downstream; absent → one generated + forwarded); `registry_down_tags_unresolved` (matcher returns Unresolved → header `unresolved`, still forwarded). Degraded mode: no DiscoverySink configured → unknown tagged `cand_` + forwarded, not enqueued (documented).
- [ ] Run → implement → run.
- [ ] Commit `feat(http): discovery-lane feed for HTTP unknowns + idempotency-key contract + outage tagging`.

### Task 4: `[http_proxy]` config + serve() wiring + graceful shutdown

**Files:** `crates/deblob/src/config.rs` (+ `[http_proxy]`), `crates/deblob/src/serve.rs` (wire), `deblob.example.toml`.

**Produces:** `[http_proxy]` config (serde, `deny_unknown_fields`): `enabled: bool` (default false), `listen_addr`, `upstream_allowlist`, `route`, `max_body_bytes`, `max_header_bytes`, timeouts, `require_auth`. Any ingest auth token env-only. Wired into `serve()`: when enabled, spawn `HttpProxy::run` sharing the `HotMatcher` + discovery sink + `CancellationToken`, drained in graceful shutdown after the management API. Disabled = nothing spawned, prior behavior byte-identical.

- [ ] Tests: `config_parses_http_proxy_section` (+ defaults when absent; deny_unknown_fields rejects typos); `disabled_by_default_spawns_no_proxy` (a serve-level wiring assertion — enabled=false → no proxy constructed); `enabled_spawns_proxy` (enabled=true → proxy wiring constructed). Keep it non-Docker where possible.
- [ ] Run → implement → run.
- [ ] Commit `feat(bin): [http_proxy] config + HttpProxy wired into serve (disabled by default)`.

## Task order

1 → 2 → 3 → 4. Reviewer checkpoint after Task 2 (hardening is the security surface). Each task: fresh implementer + independent reviewer.

## Deferred

TLS client-cert enforcement beyond a bearer/API-key option (P3); HTTP/2 or streaming bodies beyond the bounded model; any payload transformation.
