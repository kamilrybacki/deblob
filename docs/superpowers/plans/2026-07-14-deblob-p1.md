# Deblob P1 (Deterministic Core) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the P1 deterministic core of Deblob — a transactional Kafka relay that fingerprints every JSON message, tags it with an immutable schema identity in headers, persists schemas in a Redis vault with atomic publication, and quarantines malformed data. No SLM in P1.

**Architecture:** Cargo workspace of small crates around vendor-free ports in `deblob-core`. Hot path: consume → strip reserved headers → bounded parse → structural fingerprint → LRU/bucketed-index match → tag → transactional produce (tagged + discovery) → offsets in transaction. Cold lane (P1 subset): consume `deblob.discovery`, cluster shapes into provisional profiles via the schema monoid, mint `cand_` identities, expose candidates for manual promotion through an authenticated management API that performs atomic Lua publication into Redis.

**Tech Stack:** Rust (edition 2021, MSRV 1.80+), tokio, rdkafka 0.37 (features `cmake-build`, `tokio`, `ssl`), redis 0.27 (`tokio-comp`), axum 0.8, serde_json (`arbitrary_precision`), sha2, data-encoding (base32), uuid (`v7`), lru, proptest, testcontainers-modules (`kafka`, `redis`), cargo-fuzz, tracing + tracing-subscriber, prometheus.

**Spec:** `docs/superpowers/specs/2026-07-14-deblob-design.md` — every requirement referenced below by section (§).

## Global Constraints

- Payload bytes are NEVER mutated; tags ride exclusively in Kafka headers (§2).
- Exactly one `deblob-schema-id` header on output; ALL inbound `deblob-*` headers stripped first (§3.2, §9).
- Header values are IDs only — never schemas, reasons, or model output (§3.2).
- Hash preimage: `sha256("deblob-canon-v1\0" || canonical_bytes)`; IDs `sch_<base32-lowercase-nopad(digest)>` (§4).
- Numbers never round-trip through f64; `1`/`1.0`/`1e0` structurally equivalent; NaN/±Inf = malformed (§4).
- Duplicate JSON object keys = malformed → quarantine (§4).
- All parse limits enforced BEFORE tree allocation (§4).
- Redis: refuse to start without AOF persistence unless `--unsafe-volatile`; runtime persistence failure freezes promotions + fails readiness (§6).
- `deblob:schema:*` write-once; collision byte-compare, mismatch = process-fatal error (§6).
- Publication (schema + family version + index + alias + audit) is ONE atomic Lua script (§6).
- Aliases write-once, acyclic (§5, §6).
- Redis unavailable on hot path → tag `unresolved`, never `cand_` (§10).
- Kafka: cooperative-sticky, abort txn on revoke, tombstones pass through with reserved tag, partition p→p (§3.2).
- Replays mint identical tags — no fresh IDs during replay (§3.2).
- Management API on separate port from any ingest listener; bearer auth from env; promotions audited (§8).
- No payload contents in logs by default (§9, §11).
- Conventional commits; TDD; every crate ≥80% coverage.

```rust
use data_encoding::BASE32_NOPAD;

#[derive(Debug, thiserror::Error, PartialEq, Eq)]
pub enum IdError {
    #[error("wrong prefix, expected {expected}")]
    WrongPrefix { expected: &'static str },
    #[error("invalid base32 body")]
    InvalidBody,
}

macro_rules! digest_id {
    ($name:ident, $prefix:literal) => {
        #[derive(Debug, Clone, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
        #[serde(transparent)]
        pub struct $name(String);

        impl $name {
            pub fn from_digest(digest: &[u8; 32]) -> Self {
                Self(format!("{}{}", $prefix, BASE32_NOPAD.encode(digest).to_ascii_lowercase()))
            }
            pub fn parse(s: &str) -> Result<Self, IdError> {
                let body = s.strip_prefix($prefix).ok_or(IdError::WrongPrefix { expected: $prefix })?;
                let up = body.to_ascii_uppercase();
                let bytes = BASE32_NOPAD.decode(up.as_bytes()).map_err(|_| IdError::InvalidBody)?;
                if bytes.len() != 32 { return Err(IdError::InvalidBody); }
                Ok(Self(s.to_string()))
            }
            pub fn as_str(&self) -> &str { &self.0 }
        }
    };
}
digest_id!(SchemaId, "sch_");
digest_id!(CandidateId, "cand_");

#[derive(Debug, Clone, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
#[serde(transparent)]
pub struct FamilyId(String);
impl FamilyId {
    pub fn new_v7() -> Self { Self(format!("fam_{}", uuid::Uuid::now_v7())) }
    pub fn parse(s: &str) -> Result<Self, IdError> {
        let body = s.strip_prefix("fam_").ok_or(IdError::WrongPrefix { expected: "fam_" })?;
        uuid::Uuid::parse_str(body).map_err(|_| IdError::InvalidBody)?;
        Ok(Self(s.to_string()))
    }
    pub fn as_str(&self) -> &str { &self.0 }
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, serde::Serialize, serde::Deserialize)]
pub struct FamilyVersion(pub u32);

#[derive(Debug, Clone, PartialEq, Eq)]
pub enum SchemaRef {
    Known(SchemaId),
    Provisional(CandidateId),
    Unresolved,
    Malformed,
    Tombstone,
}
impl SchemaRef {
    pub fn header_value(&self) -> String {
        match self {
            SchemaRef::Known(id) => id.as_str().to_string(),
            SchemaRef::Provisional(id) => id.as_str().to_string(),
            SchemaRef::Unresolved => "unresolved".into(),
            SchemaRef::Malformed => "malformed".into(),
            SchemaRef::Tombstone => "tombstone".into(),
        }
    }
}
```

Hand-rolled recursive-descent JSON parser over `&[u8]` (do NOT delegate to serde_json — we need duplicate-key rejection, number-text preservation, and limit checks during the walk, spec §4). Size check first (`bytes.len() > limits.max_bytes` → `SizeExceeded` before any allocation). Track depth on every `[`/`{`; check key length at key parse; check field count per object; collect duplicate keys via sorted insert into `Vec<(String, Node)>` with binary-search dup detection; numbers captured as the raw byte slice → `String` after validating JSON number grammar (rejects NaN/Inf — they are not valid JSON tokens; a bare `NaN` fails as `ParseError`, satisfying `NonFiniteNumber` semantics via serde-free grammar). UTF-8 validation on string content → `Utf8Error`. Arrays inspect at most `max_array_inspect` elements then skip remaining values with a value-skipper (still depth/size-guarded), returning `truncated=true`.

~180 lines; the parser is the crate's security boundary — every branch tested above, fuzzed in Task 13.
