Deblob: Emergent Schema Identification for Event Streams
via a Deterministic Structural Core and a Trust-Gated Language-Model Lane

Kamil Rybacki

with Claude (Anthropic) and Hermes — homelab agent collaboration

Technical whitepaper · July 17, 2026 · github.com/kamilrybacki/deblob

Abstract

Event streams accumulate schema "blobs": producers emit JSON whose structure drifts, forks, and multiplies until no one can say how many distinct schemas a topic actually carries. Deblob tags every event with the schema family it belongs to and discovers genuinely new schemas, under one hard invariant: two different schemas are never merged. The system splits the problem into a deterministic structural core (canonical fingerprinting, algebraic profiling, and nearest-neighbour retrieval, running at sub-millisecond per-event cost inside an exactly-once Kafka relay at 598 records/s on modest hardware) and a small-language-model (SLM) lane that handles the semantic middle ground where structure alone is provably insufficient. A deterministic trust gate ensures no model proposal is ever applied without independent structural corroboration. We fine-tuned a 0.5-billion-parameter model on 720 synthetic, ground-truth-labelled examples for under $0.10 of GPU time; with completion-only loss, balanced sampling, and finite-hypothesis scoring, it identifies never-before-seen schema families with 89% recall on held-out families — without once mistaking a new family for an existing schema — while selecting only catalogued schema identifiers, making identifier hallucination structurally impossible. The full loop — feedback export, remote training, governed promotion — was exercised end-to-end on production infrastructure. We report the architecture, data representation, training method, and honest limitations.

Keywords: schema inference, event streams, exactly-once processing, small language models, LoRA fine-tuning, selective prediction, human-in-the-loop

1 Introduction

A production event bus is rarely the tidy place its documentation claims. Teams ship new producers, rename fields, wrap payloads in envelopes, and re-use generic field names (id, status, amount) for unrelated things. The result is a stream whose effective schema catalogue is unknown: a blob. Downstream consumers break on drift they cannot see; data teams cannot answer "how many event types do we actually have?"

Deblob answers that question continuously. For every event it decides one of three things: the event matches a known schema family (and how: exactly, as compatible drift, or as a structurally similar but semantically incompatible lookalike), it belongs to a new family never seen before, or the evidence is insufficient and the system abstains, deferring to a human. Two properties are non-negotiable: the hot path must be fast and deterministic, and the system must never falsely merge two different schemas — a merge poisons every downstream consumer and is far costlier than a deferred decision.

The central design claim is that these goals need two different tools. Structure is cheap and reliable: canonical fingerprints and structural distance settle most events in microseconds. But we show empirically (§6.4) that structural distance cannot separate "genuinely new schema" from "same schema, renamed fields" — the two populations overlap. That residual is a semantic judgement, and we delegate it to a small fine-tuned language model that is never trusted alone: a deterministic gate must corroborate every proposal before anything is applied. The outputs are concrete: every event republished with its schema-family tag, a continuously maintained schema catalogue, and a short review queue holding only the cases no layer could decide safely.

2 System Design

2.1 Deterministic core

Events are parsed with bounded, fuzz-hardened parsing, reduced to a canonical structural fingerprint (key order, whitespace, and value content do not affect it), and folded into a per-family monoid profile — an algebraic summary of field paths, types, nullability, and observation counts that composes associatively, so profiles of event batches merge without re-scanning history. Candidate schema families are retrieved by structural distance (a weighted-Jaccard overlap measure on the profile: shared fields, weighted, over all fields). The relay consumes from Kafka and republishes tagged events under exactly-once transactional semantics (a crash or retry can never emit a duplicate or drop an event); batching plus intra-batch produce pipelining raised throughput 43× (from 14 to 598 records/s) with the exactly-once guarantee preserved per batch and verified under crash, rebalance, and duplicate-delivery chaos tests. Published schemas live in a Redis vault with atomic, write-once publication.

2.2 The trust gate

The SLM lane may only propose. A proposal to match an existing schema is applied only when the deterministic evidence independently corroborates it: the proposed family is the top-ranked retrieval candidate, its structural distance is at most 0.15, the margin to the runner-up is at least 0.10, and the family has at least 20 observations. Model confidence plays no role in the decision. Everything else routes to a human. In an exhaustive test over all gate-condition combinations, no configuration exists in which an uncorroborated model proposal is applied — the no-false-merge property is structural, not statistical.

2.3 Governed continual learning

Human decisions (promotions, accepted and rejected proposals) are captured as immutable, provenance-bearing training examples — with reason codes, so a rejection caused by a policy denial is not mislabelled as a model error. Retraining runs remotely from a stable base checkpoint; a candidate model must pass a statistical offline gate (zero false merges on a held-out corpus, per-family precision floors with Wilson confidence bounds, non-inferiority versus the active model) and a live-shadow hold before an explicit, separate promotion step activates it. The training hook returns artifact digests only; by construction it cannot move the active-model alias. Full-bundle rollback (weights, tokenizer, prompt template, quantization, retrieval index) is retained.

3 Input Data and Schema Representation

Deblob ingests ordinary JSON events. A representative GitHub Archive event:

{ "id": "2489651045", "type": "PushEvent", ← never shown to the model "actor": {"id": 91, "login": "octocat"}, "repo": {"id": 3, "name": "octocat/hello-world"}, "payload": { "push_id": 536390, "size": 1, "ref": "refs/heads/master", "head": "0e5bfe237a11f2c9…" }, "created_at": "2015-01-01T00:00:00Z" }

The model never sees raw events. Each event contributes to a redacted structural profile: field paths, types, presence and null counts, and numeric range buckets — no values. The decision input concatenates that profile with the retrieved candidates and their structural distances:

observation_count: 54 field_count: 6 path=["calibration"] types=["object"] present=54/54 path=["calibration">"offset"] types=["number"] numeric=["11-100"] path=["calibration">"unit"] types=["string"] path=["readings"] types=["array"] path=["sensor_reading_id"] types=["string"] RETRIEVED CANDIDATES (structural distance only) rank=1 family=fam_b128… schema=sch_pfxy… distance=0.286 rank=2 family=fam_8c91… schema=sch_hlxg… distance=0.444 ALLOWED schema_id SET = [sch_hlxg…, sch_pfxy…]

This representation has two roles. It is a privacy boundary — field values, secrets, and personal data cannot reach a model prompt, a log, or a training set, which a test suite enforces end-to-end — and it is the learning target: the shape statistics carry exactly the evidence a schema decision needs. Label-revealing fields (type, $schema, topic names) are stripped before inference and kept in an evaluator-only sidecar, so evaluation gold can never leak into the input.

4 Deterministic Core: Measured Performance

Retrieval alone, however, is a reckless decider: accepting every top-1 candidate on the benchmark corpus yields 100% coverage with 39 false merges in 96 events. Passing the same events through the trust gate cuts coverage to 54% and false merges to 1 (upper 95% bound 10.1%), with the remainder deferred to a human. That trade — coverage for a hard safety floor — is the system's operating philosophy: everything the gate cannot corroborate is a safe error (a review), never a dangerous one (a merge).

5 Learning the Semantic Decision

5.1 Task and corpus

The SLM's task is a bounded classification rendered as a tool call: given the redacted profile and candidates, emit match_schema (with a relation: exact, compatible_drift, or incompatible_similarity), new_candidate, or abstain. Training and evaluation use a synthetic corpus whose labels are true by construction: 90 base schema families, each expanded by known transformations — 30% exact re-encounters, 20% compatible drift (added optional fields, widened nullability, wrappers), 10% incompatible lookalikes (unit swaps, generic-name re-use — the false-merge traps), 20% genuinely new families, 20% ambiguous or low-evidence cases (900 cases total). The split is family-partitioned: all variants of a family land on the same side, so a held-out family's siblings never appear in training (720 train / 180 test).

5.2 Training method

Two training choices carry most of the improvement. First, completion-only loss: the decision prompt is hundreds of tokens and the answer roughly thirty; with a naive full-sequence objective over 90% of the gradient teaches the model to reproduce the prompt, and the first attempt duly collapsed to always answering match_schema (0/16 new families detected). Masking the prompt concentrates all learning on the decision. Second, balanced sampling: matches dominate the natural distribution (60/20/20), so minority decisions are oversampled to parity before training.

5.3 Finite-hypothesis scoring

At inference the model does not generate freely. The legal decision space is finite — each allowed schema identifier crossed with three relations, two novelty labels, three abstain causes — so Deblob enumerates every legal completion, scores each by length-normalised conditional log-likelihood under the model, and selects the argmax. On identical weights this nearly doubled decision accuracy versus free generation (30% → 55%) and makes identifier hallucination structurally impossible: an identifier outside the retrieved catalogue can never be selected, only scored against. A deterministic veto completes the policy: if the nearest known family is beyond the gate distance, the decision is forced to new_candidate regardless of the model's preference.

6.4 Why structure alone cannot finish the job

This is the paper's central negative result, and it justifies the architecture. A pure distance rule at 0.15 catches every new family but wrongly splits drifted ones; widening it to 0.40 (§6.3, v5) reverses the trade. No threshold wins both. The production policy therefore layers them: the veto handles distances beyond any plausible drift, the fine-tuned model judges the middle, and the trust gate guarantees that whatever either of them gets wrong becomes a deferral rather than a merge.

7 Limitations

  1. Synthetic ground truth. Evaluation gold is independent of the gate and hidden from the model, but it is generated by construction, not collected from production traffic. Ingestion for real corpora (GitHub Archive, Wikimedia event streams) exists; sealed evaluation on them has not yet run.
  2. The held-out set became a development set. Five iterations consulted the same held-out families, so the final point estimates are optimistically selected. A sealed audit corpus with frozen thresholds, and multi-seed runs with confidence intervals, are the required next step.
  3. Single seed, point estimates. All accuracy figures are one training trajectory.
  4. A veto's gain is not a model's gain. The 100% new-family recall of the conservative operating point belongs to the combined policy; the model alone achieves 89%. Both are reported deliberately.
  5. Zero observed false merges is not a production guarantee. It is a structural property of the gate plus an empirical observation on a bounded sample (upper 95% bound 10.1% on the accepted subset); production-scale labelled traffic is needed for a production-grade bound.
  6. Pre-alpha software. Single maintainer, no release history; some documentation still describes earlier states of the system.

8 Conclusion

Deblob demonstrates that schema identification for event streams can be split into a fast, provably safe deterministic core and a small, cheaply trained language model whose authority is bounded by construction. The deterministic lane tags events in 0.43 ms and moves them exactly-once at 598 records/s; the learned lane, trained for under $0.10, identifies unseen schema families at 89% recall — never once mistaking one for an existing schema — while structurally unable to hallucinate identifiers or force a merge. The architectural contribution is the division of authority: the model can only ever widen what the system understands, never corrupt what it knows. Continual learning then makes coverage a function of accumulated human feedback rather than model scale — the loop, from feedback capture to governed promotion, is implemented, tested against real infrastructure, and awaits production traffic.
